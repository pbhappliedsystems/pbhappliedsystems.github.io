02 — Cost Structure
Eliminate Vendor Lock-In & Per-Token Fees
Cloud LLM APIs charge per token — costs that compound with scale. Quantized open-weight models run on owned or leased GPU infrastructure at a fixed cost. The same capability that costs $100/month at scale via API can cost $5–15/month on quantized private infrastructure.
03 — Measured Performance
Evaluated Before Deployment — Not After
Every model in this demo was benchmarked using quant_eval — a proprietary evaluation framework built specifically to eliminate the guesswork of which LLM to use for which type of agent. Traditional industry benchmarks (MMLU, HellaSwag, HumanEval) measure general capability in isolation. quant_eval measures what actually matters for agentic deployment: coherence, instruction following, reasoning depth, and task completion — scored against the model's own full-precision F16 baseline, not an abstract leaderboard.
Before a single line of agent code is written, quant_eval identifies each model's capabilities and failure modes under the exact output format, reasoning depth, and instruction complexity required by the intended agent template. A model that scores 0.97 on instruction following maps to the code agent. One that scores 0.94 on reasoning maps to the reasoning agent. Quantization-induced degradation is measured precisely — so deployment decisions are made on evidence, not assumption.

Part I — Quality-Gated Generation
01 — Quality Parity
Same Quality, Local or Cloud
Across production runs, lexical diversity (MATTR ≈ 0.77) and instruction-output fidelity held within half a percentage point whether generation ran on an RTX 4090 or a cloud serverless GPU — because the quality layer is identical and substrate-independent.
02 — Throughput & Cost
~13× Faster, Cents per Thousand
AWQ-vLLM cloud generation runs at ~3,300 records/hour — roughly 13× local — at ~$0.45 per 1,000 fully audited records, the lowest cost of any serving format tested, with local generation 26–52× cheaper than mid-tier and frontier APIs.
03 — Attested Compliance
Audited, Not Asserted
Every dataset ships with a machine-readable report — rejection reasons, diversity, placeholder coverage, reconciliation, and an independently audited residual PII leak rate-driven and is scanner-verified.
Part II — Construction-True De-identification (Addendum)
01 — Construction-True Labels
Correct by Construction, Not Inferred
An injector places synthetic PII into the text and logs the exact span it wrote, so the labels are ground truth by construction — with no detector recall ceiling to inherit, and no real personal data to re-identify, making the dataset safe to sell into regulated settings.
02 — Learned-NER Premium Tier
Free-Text Recall, Recovered from Zero
A gap-only GLiNER2 hybrid recovers personal-name and institution-ID recall from 0.00 to 0.90+ at an overall F1 of 0.958–0.965 — a strict gain over the regex base, with the model's scope established by measurement, never by assumption.
03 — Hardened & Provable
Sealed Delivery, Restored Economics
A template-leak rejection gate, a schema-1.2 run manifest, and a SHA-256 checksum-sealed package make every claim auditable — while a diagnosed-and-resolved substrate fault restored steady-state cloud cost to a measured ~$0.47 per thousand records.

The methodology is the product. Frontier APIs generate raw tokens; SynthEval turns generation — on any substrate — into a compliant, attested deliverable a regulated buyer can defend, with labels correct by construction and quality measured rather than asserted.
